## FIX 1: 승인 대기 카드 선택 시 하이라이트 불일치

**파일**: `PendingPage.xaml`
**증상**: 카드를 클릭하면 밝아지는 영역(ListViewItem 기본 선택 효과)이 카드 Border와 위치/크기가 맞지 않음
**목표**: 선택 하이라이트 영역이 카드 Border와 정확히 일치하도록 수정. ListViewItem의 기본 패딩/마진을 제거하고, 아이템이 전체 너비로 Stretch되게 할 것.

```
커밋: fix(ui): 승인 대기 카드 선택 하이라이트 영역 정렬
```

## FIX 3: 승인 대기 카드 크기 20% 확대

**파일**: `PendingPage.xaml`
**증상**: 카드 내 텍스트와 여백이 작아 가시성 부족
**목표**: 카드의 전체 크기를 약 20% 키움. 구체적으로:
- 카드 내부 Padding 약 20% 증가
- 신청자 이름, 연락처, 장비명, 신청일시 등 모든 텍스트의 FontSize 약 20% 증가
- 기간 배지, 액세서리 배지의 Padding과 FontSize도 약 20% 증가
- 카드 간 간격(Margin)도 비례 증가

```
커밋: fix(ui): 승인 대기 카드 크기 및 텍스트 20% 확대
```

## FIX 5: 라이트 모드 전환 시 어두운 초록색이 민트색으로 안 바뀜

**증상**: 다크 모드의 진한 초록(#004D40, #00695C, #004246, #1B5E20 등)이 라이트 모드로 전환해도 그대로 유지됨
**원인**: 하드코딩된 색상값이 ThemeResource를 거치지 않고 직접 사용됨
**목표**: 
1. 프로젝트 전체 XAML 파일에서 초록 계열 하드코딩 색상을 검색하라 (`grep -rn` 활용)
2. 모든 하드코딩 색상을 ThemeResource 브러시로 교체하라
3. `Themes/BrandColors.xaml`에 해당 브러시가 Light/Dark 양쪽 ThemeDictionaries에 정의되어 있는지 확인하고, 없으면 추가하라
4. 라이트 모드 대응 컬러:
   - 다크 `#004D40` ↔ 라이트 `#B2DFDB`
   - 다크 `#00695C` ↔ 라이트 `#00897B`
   - 다크 `#1B5E20`(배지 배경) ↔ 라이트 `#E8F5E9`
   - 다크 `#A5D6A7`(배지 텍스트) ↔ 라이트 `#2E7D32`

```
커밋: fix(ui): 라이트/다크 모드 전환 시 브랜드 컬러 정상 반영
```
