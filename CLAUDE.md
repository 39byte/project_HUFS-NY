## 스텝 2: "대여 현황" 탭 캘린더 레이아웃 변경

### 목표
캘린더가 사이드바를 제외한 콘텐츠 영역 전체를 채우는 현재 레이아웃을 **정사각형 박스 형태**로 변경하여 콘텐츠 영역 **중앙**에 배치한다.

### 상세 요구사항
- 캘린더의 **세로 길이**: 앱 창 세로 길이보다 약간 작게 (여백 확보)
- 캘린더의 **가로 길이**: 세로 길이와 동일 (정사각형 비율 유지)
- 캘린더를 감싸는 컨테이너는 콘텐츠 영역에서 **수평 중앙 정렬**
- **사이드바(NavigationView)가 접혀 있을 때도** 동일한 정사각형 중앙 배치 유지
- 캘린더 내부 콘텐츠(월 타이틀, 요일 헤더, 날짜 셀, 대여 바 등)는 기존 기능 그대로 유지

### 구현 방향
- CalendarPage.xaml에서 캘린더 영역의 Width/Height 바인딩 또는 SizeChanged 이벤트로 정사각형 비율 계산
- HorizontalAlignment="Center" 적용
- ViewModel 코드는 수정하지 않는다

### 커밋 메시지
`[UI] 대여 현황 캘린더를 정사각형 박스 형태로 중앙 배치`

## 스텝 4: "대여 신청" 탭 연락처 입력칸 3분할

### 목표
현재 단일 TextBox인 연락처 입력 필드를 **3개의 TextBox로 분할**하여, 전화번호 앞자리 / 중간자리 / 뒷자리를 각각 입력하도록 변경한다.

### 상세 요구사항

**레이아웃:**
- 3개의 TextBox를 가로로 나란히 배치
- 각 TextBox 사이에 **"-"(대시)** 텍스트를 표시 (TextBlock 등으로)
- 전체 구조: `[앞자리] - [중간자리] - [뒷자리]`

**PlaceholderText:**
- 첫 번째 칸: `010`
- 두 번째 칸: `0000`
- 세 번째 칸: `0000`

**입력 제한:**
- 첫 번째 칸: 최대 3자리, 숫자만 입력 가능
- 두 번째 칸: 최대 4자리, 숫자만 입력 가능
- 세 번째 칸: 최대 4자리, 숫자만 입력 가능
- 숫자 외 문자 입력 시 무시 (TextChanging 또는 BeforeTextChanging 이벤트로 필터링)

**Tab 키 이동:**
- 각 TextBox 간 Tab 키로 자연스럽게 포커스 이동 가능하도록 TabIndex 설정
- 단, 이 기능 구현이 기존 레이아웃에 큰 변형을 가져올 경우 skip 가능

**ViewModel 연동:**
- RentalFormViewModel의 `Contact` 프로퍼티에 최종 값을 전달할 때, 3개 필드를 "-"로 합쳐서 전달 (예: `010-1234-5678`)
- ViewModel의 기존 Contact 프로퍼티 타입(string)은 유지
- 합치는 로직은 code-behind(XAML.cs) 또는 XAML 바인딩 컨버터로 처리 — ViewModel에 새 프로퍼티를 추가하지 않는 방향을 우선 시도하되, 구현이 까다로울 경우 ViewModel에 `ContactPart1`, `ContactPart2`, `ContactPart3` 프로퍼티를 추가해도 무방

### 커밋 메시지
`[UI] 연락처 입력칸을 전화번호 3분할 구조로 변경`
